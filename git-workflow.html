Git + VS Code Workflow Cheat Sheet

Edit File
(VS Code)
→
Stage File
(git add)
→
Commit Changes
(git commit)
→
Push to GitHub
(git push)
→
GitHub Repo

Common Commands

# Stage all changes
git add .

# Stage a specific file
git add filename.ext

# Commit staged files
git commit -m "Commit message"

# Push changes to GitHub
git push origin main

# Pull latest updates
git pull origin main

# Check repo status
git status

# Check commit history
git log --oneline

# Check remote URL
git remote -v

Tips

  • Always save files in VS Code before staging/committing.
  • Use descriptive commit messages for easier tracking.
  • Stage only the files you want to commit.
  • Keep commits small and focused.
  • Use Live Server to preview changes instantly.
